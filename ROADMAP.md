# Loft — роадмап (порезан владельцем 2026-07-15)

## Правило границы (главный итог обсуждения)

**Скрипт в ядре loft — только детерминированный конвертер данных
(вход → выход): convert.py, fix_tables.py, link_check.py. Всё.**
Проверки смысла (терминология, полнота, санитария, трассировка) —
УКАЗАНИЯ агентам в скиллах: агент читает скилл и делает это сам.
Функции с состоянием, историей и UI (git/дифф/версии/граф-экран/
канбан) — фичи Specweave v2, не ядра. Никаких эмбеддингов, векторов,
MCP, демонов, расписаний — никогда.

## Loft — что делаем

- [x] **Экспорт в PDF** (сделано в 1.0.0) — единственный экспортный формат ядра:
  скилл deliver-pdf, pandoc md→pdf, резолв [[wikilinks]] в текст,
  mermaid — как получится без тяжёлых зависимостей. docx/doc — НЕ
  делаем (решение владельца: глупо). Confluence-экспорт — НЕ делаем:
  владелец импортирует .md в Confluence сам; в скилл — только указание
  «перед импортом резолвни/проверь wikilinks и картинки».
- [x] **review-intake** (сделано в 1.0.0) — разбор возврата заказчика (входящая труба,
  не экспорт): pandoc --track-changes=all для правленого docx (сейчас
  правки молча схлопываются — риск №1), таблица решений
  «правка → принять/отклонить/вопрос». Пара правок — без церемонии,
  полный раунд — stage.
- [x] **Указания в скиллах вместо скриптов** (tz-audit: синонимы; сделано) (глоссарий, линт имён,
  светофор, REQ-трассировка): усилить формулировки в tz-audit
  (синонимы одной системы против глоссария — ИСОБ/ISOB/OB), tz-write
  (термин не из глоссария → сначала в глоссарий), verifier (санитария:
  IP/хосты/токены в тексте). Агенты делают это чтением и grep'ом по
  ходу работы — без новых скриптов.
- [x] **templates.md в tz-elicit** (сделано в 1.1.0) — банк формулировок требований
  (EARS по-русски, user story + Given/When/Then, НФТ-таблица). Это
  данные для агента, не код.
- [x] **knowledge-map** (сделано в 1.1.0) — воспроизводимая карта
  корпуса скиллом уровня stage (scout'ы, без инфраструктуры) — по
  образцу ручного _KNOWLEDGE-MAP.md в iyb.
- [x] **Конвертер: --unroll-pre** (сделано в 1.1.0) — JSON-таблицы iyb →
  GFM + код-фенсы после таблицы, опциональным флагом.
- [ ] **Конвертер: человекочитаемые имена вложений** — переименование
  `2923549_*.pdf` по алиасам с переписью ссылок; внутри существующих
  детерминированных скриптов, по запросу.
- [x] **golden corpus в репо** (test/fixtures + 40+ тестов; сделано в 1.0.0) (dev-инфраструктура): регресс-тест
  конвертера на синтетической мини-выгрузке при сборке архива —
  защищает от порчи 446 страниц при правках ядра. В проекты не
  устанавливается.
- [ ] **Plugin-упаковка** — решение владельца 2026-07-15: пока не делаем; вернуться, когда появится второй пользователь ядра.
