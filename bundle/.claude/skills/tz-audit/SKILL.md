# tz-audit — аудит документации

Используй для: приёмки чужого ТЗ, ревизии своего перед сдачей,
периодической проверки корпуса, входного анализа нового проекта
(«что нам досталось»).

## Оси проверки

1. **Полнота** — против `spec/_STRUCTURE.md` / профиля из
   `memory/structures/`: какие обязательные разделы пусты или
   отсутствуют; у требований есть ли критерии приёмки.
2. **Противоречия** — внутри документа; между документом и корпусом
   (глоссарий, соседние ТЗ, wiki); дубли, описывающие одно по-разному;
   разнобой имён одной системы (синонимы вида ИСОБ/ISOB/OB) — сверь
   grep'ом по корпусу против канонического имени из глоссария.
3. **Целостность** — skill `link-check`: битые `[[wikilinks]]`,
   `![[embeds]]`, относительные ссылки; страницы-сироты.
4. **Обоснованность** — утверждения без источника и без пометки
   допущения; «висящие» открытые вопросы, которых нет в `QUESTIONS.md`.
5. **Актуальность** — расхождение с источником истины: дата снапшота
   `wiki/` против живого Confluence; версии API/систем в тексте против
   текущих; упоминания «планируется» с прошедшей датой.
6. **Гигиена** — секреты и внутренние адреса в открытом тексте
   (IP, хосты, токены — правило 8 контракта); мусорные дубли файлов.

## Отчёт

Светофор по каждой оси × каждому документу (или разделу — для
одиночного дока): 🟢 ок / 🟡 замечания / 🔴 блокер. Формат:

- одиночный документ → секция «Аудит» в ответе владельцу + находки
  в `BACKLOG.md`;
- корпус → это `stage`: параллельные scout'ы по доменам, сводный
  `stages/NNN-audit-<slug>/report.md` со светофорной таблицей,
  каждая находка — строка в `BACKLOG.md` вида
  `P1 | документ | что не так | ev:путь:строка | src:audit-NNN`.

## Правила

- Каждая находка несёт свидетельство (`ev:` путь и строка) — находка
  без ev не публикуется.
- Дедупликация против уже открытых находок в `BACKLOG.md` — прежде
  чем писать, проверь, нет ли такой.
- Пока открыто 20+ находок аудита — закрывай их, новый аудит не
  начинай (правило 5 контракта).
- Аудит не чинит: найденное — в очередь, исправления — отдельной
  работой (мелкие — сразу вторым уровнем, большие — stage).
