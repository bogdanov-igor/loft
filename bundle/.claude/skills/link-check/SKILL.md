# link-check — целостность ссылок

```sh
python3 .claude/skills/link-check/scripts/link_check.py wiki spec
python3 .claude/skills/link-check/scripts/link_check.py spec --no-orphans
```

Скрипт (stdlib, детерминированный) собирает по указанным каталогам:

- `[[target]]` / `[[target|текст]]` / `[[target\|текст]]` / `![[target]]` —
  разрешение по basename в стиле Foam/Obsidian (без регистра, NFC) по
  всему корпусу;
- относительные `[текст](путь)` и картинки `![alt](путь)` —
  существование файла (http/mailto/# пропускаются);
- сирот: md-страницы без единой входящей ссылки (служебные `_*`,
  `Home.md`, `README.md`, `MEMORY.md` не считаются).

Строки внутри код-фенсов (```/~~~) не проверяются.

Выход: сводка (всего ссылок / битых / сирот) и список
`файл:строка → цель` по каждой битой. Код возврата 1 при битых —
можно ставить в пайплайн.

## Правила

- Битая ссылка в сгенерированной `wiki/` — дефект конвертера: чинится
  в скриптах `ingest-confluence`, не правкой wiki руками.
- Битая ссылка в `spec/` — чинится сразу (второй уровень, без
  церемонии), либо строкой в `BACKLOG.md`, если целей много.
- Сирота — не всегда дефект (свежий документ ещё не подключён), но
  сирота старше пары недель — кандидат в `tz-audit` находку.
- Прогон обязателен: после `ingest-*`, после переименований файлов,
  перед вердиктом verifier по документу с новыми ссылками.
