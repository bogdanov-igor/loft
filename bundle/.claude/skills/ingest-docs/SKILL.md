# ingest-docs — входящие файлы

Любой прилетевший файл сначала кладётся в `inbox/` как есть.
Оригиналы не удаляются: конвертация ссылается на оригинал, и любое
сомнение решается по нему.

## docx / doc — детерминированно

```sh
# из каталога spec/: медиа ляжет в spec/assets/media/, и ссылки в
# документе будут корректны относительно него
cd spec && pandoc -f docx -t gfm --wrap=none --extract-media=assets \
  "../inbox/<файл>.docx" -o "<слаг>.md"
```

`.doc` (старый формат) pandoc не читает — сконвертируй в `.docx`
(`soffice --headless --convert-to docx`, LibreOffice) или попроси
владельца пересохранить. После конвертации: проверь таблицы и
картинки глазами, добавь frontmatter (`source: inbox/<файл>.docx`,
`ingested: <дата>`), свяжи `[[wikilink]]`-ами с корпусом.

## pdf — конспект с пометкой

PDF читается напрямую (Read, ~20 страниц за вызов; большие — частями
или параллельными scout'ами). Результат — **конспект, а не копия**:
структурированный md с обязательной шапкой:

```markdown
> Конспект. Оригинал: [[<файл>.pdf]] (N страниц, конспект от <дата>).
> Числа и формулировки при использовании сверять с оригиналом.
```

Таблицы из PDF переноси как GFM; страницы оригинала указывай у
ключевых фактов (`стр. 12`). Это единственное место ingest'а, где
работает модель, а не скрипт — поэтому пометка обязательна.

## Картинки (скриншоты, схемы)

В `assets/`, в документ — `![[файл.png]]` с подписью: что изображено,
откуда, от какой даты. Схема с картинки нужна как живая диаграмма →
перерисуй в ```mermaid рядом с embed'ом, пометив «перерисовано с
[[оригинала]]» — картинка остаётся источником истины.

## html (не Confluence-выгрузка)

Одиночные html-страницы: `pandoc -f html -t gfm --wrap=none`, дальше
как docx. Целая выгрузка Confluence — это skill `ingest-confluence`.

## Правила

- Каждый сконвертированный документ: frontmatter с `source:` и
  `ingested:`, ссылки в/из корпуса (иначе он сирота — `link-check`
  покажет).
- Секреты и внутренние адреса из входящих в корпус не переносятся —
  плейсхолдеры (правило 8 контракта); оригинал в `inbox/` остаётся
  как есть.
- Пачка файлов → обработка параллельными сабагентами, по одному файлу
  на агента; итоги сведи в один список «что пришло → где лежит».
