# ingest-confluence — Confluence → wiki/

Конвертер детерминированный (pandoc + lxml): LLM не переписывает
факты. Зависимости: `pandoc`, `python3` + `lxml`.
Скрипты — в `scripts/` рядом с этим файлом.

## Первичная конвертация / обновление снапшота

1. Выгрузка: Confluence → Space Settings → Export → HTML. Zip положи
   в `inbox/`.
2. Распакуй во временный каталог (не в корпус): каталог спейса
   содержит `index.html` и страницы `*.html`.
3. Конвертируй:
   ```sh
   python3 .claude/skills/ingest-confluence/scripts/convert.py <raw>/<SPACE> wiki \
       [--space KEY] [--base-url https://confluence.example.com]
   python3 .claude/skills/ingest-confluence/scripts/make_index.py wiki
   ```
   `--space` — ключ спейса (дефолт: имя каталога выгрузки);
   `--base-url` — адрес Confluence для `source:` во фронтматтере и
   ссылки «Источник» в футере (без флага эти строки не пишутся).
   Пример: `--base-url https://confluence.<домен-компании>` (реальный
   адрес возьми из `source:` существующих страниц wiki проекта).
4. Прогони skill `link-check`; изучи лог fallback'ов конвертера
   (таблицы, оставшиеся HTML, с причинами) — их должно быть мало.
5. Дата снапшота берётся из подвала `index.html` выгрузки и пишется
   строкой «Снапшот Confluence: YYYY-MM-DD» в `wiki/Home.md`;
   расхождение снапшота с живым Confluence — ось skill'а `tz-audit`.

Повторная выгрузка перезаписывает страницы Confluence в `wiki/` и
удаляет исчезнувшие из выгрузки (`[stale-removed]` в логе). Файлы БЕЗ
`confluence_id` во фронтматтере (рукописные слои `_TRAINING/`,
`_specs/`, `_KNOWLEDGE-MAP.md` и пр.) конвертер не трогает никогда.
Поэтому правки смысла в страницах wiki руками запрещены (контракт,
«Правда живёт в файлах»): они умрут при следующем обновлении.
Комментарии и выжимки — в `spec/` со ссылками на страницы wiki.

Отчёт изменений при повторной конвертации (когда прежний
`.pagemap.json` существовал и есть отличия):

- `wiki/_CHANGES-<дата-снапшота>.md` — человеческий отчёт: секции
  Добавлено/Изменено/Перемещено/Удалено (живые страницы — wikilinks,
  удалённые — именами) и «Возможно устарели в spec/» — документы
  `spec/` (сиблинг `wiki/`), ссылающиеся на изменённые/удалённые
  страницы. Изменения ловятся по sha1-хэшу содержимого в
  `.pagemap.json` — ложных срабатываний от повторного прогона нет.
- `wiki/.ingest.json` — машинный отчёт, пишется при КАЖДОЙ конвертации:
  snapshot_date, space, base_url, pages_written, added/changed/moved/
  removed, table_fallbacks, missing_assets.

Указание агенту: изменённые страницы, на которые ссылается `spec/`
(секция «Возможно устарели в spec/»), — кандидаты в `BACKLOG.md`;
конвертер сам в BACKLOG не пишет.

## Постпроцессинг уже сконвертированной wiki

Для wiki, собранной старым конвертером (сырые HTML-таблицы, пустые
ссылки-вложения, HTML-якоря с атрибутным шлаком):

```sh
python3 .claude/skills/ingest-confluence/scripts/fix_tables.py wiki --dry-run  # посмотреть
python3 .claude/skills/ingest-confluence/scripts/fix_tables.py wiki            # применить
```

Что делает (доводит старую wiki до вывода текущего конвертера):
HTML-таблицы → GFM; пустые якоря-вложения получают имя; декоративные
иконки (`viewavatar`, эмотиконы) выпиливаются; простые `<a>` в потоке
становятся markdown/wikilinks (`.md` → `[[база|текст]]`, вложения →
`[[файл|имя]]`, внешние → `[текст](url)`); у оставшегося HTML
(fallback-таблицы) атрибуты чистятся по вайтлисту.

Флаг `--unroll-pre` (есть и у `convert.py`): таблицы с многострочными
JSON/XML-примерами не остаются HTML-fallback'ом, а разворачиваются —
ячейка получает «см. Пример N ниже», код выносится после таблицы
фенсами с языком; содержимое примеров — байт-в-байт. Меняет вёрстку
страницы, поэтому опционален.

Флаг `--expand-spans` (есть и у `convert.py`): таблицы с объединёнными
ячейками (colspan/rowspan) не остаются HTML-fallback'ом, а
разворачиваются в GFM — rowspan повторяет значение на каждой строке,
colspan дополняет пустыми ячейками. Потеря вёрстки, не данных;
опционален по той же причине.

Идемпотентен: повторный прогон ничего не меняет. Фенсы, инлайн-код и
экранированные литералы неприкосновенны; остальной текст — байт-в-байт.
После — `link-check` и сверка счётчиков wikilinks до/после (скрипт
печатает `[wikilinks] N -> M`).

## Что делает конвертация (v2)

- Дерево страниц зеркалится в папки; `pageId → файл` в `wiki/.pagemap.json`;
  ключ/название спейса и дата снапшота — в `wiki/.space.json` (их читает
  `make_index.py` для шапки `Home.md`). Ядро проектно-нейтральное: никаких
  зашитых названий спейсов/хостов, всё — через флаги и `index.html` выгрузки.
- Макросы Confluence: code/expand/info/note/warning/status/mermaid —
  в их md-эквиваленты; ссылки → `[[wikilink|текст]]`; картинки →
  `assets/` + `![[embed]]`; файлы → `attachments/` + `[[файл|имя]]`.
- Таблицы → GFM собственным писателем (не pandoc): списки в ячейках →
  строки через `<br>`, однострочный `<pre>` → инлайн-код. HTML-fallback
  только для colspan/rowspan, вложенных таблиц и многострочного кода —
  каждый случай логируется с причиной.
- Ссылки-вложения без текста получают имя из
  `data-linked-resource-default-alias` / `aria-label` / имени файла.
- Атрибутный шлак Confluence (class/style/rel/data-*) чистится по
  вайтлисту до pandoc — ссылки в потоке становятся markdown/wikilinks,
  а не сырым `<a class=...>`; jira-аватарки и эмотиконы выпиливаются;
  автолинки `<url>` с pageId резолвятся в `[[wikilink]]`.
- Страницы, исчезнувшие из выгрузки, удаляются при повторной конвертации
  (`[stale-removed]`); файлы без `confluence_id` — неприкосновенны.
