# review-intake — возврат заказчика

Используй, когда ТЗ вернулось с правками. Критично: обычная конвертация
(`ingest-docs`) МОЛЧА ПРИНИМАЕТ все track changes и ТЕРЯЕТ комментарии —
для возврата она запрещена.

## Порядок

1. **Файл — в `inbox/`**, как любой входящий. Оригинал не удаляется.
2. **Конвертация с сохранением правок** (docx с track changes):
   ```sh
   cd spec && pandoc -f docx --track-changes=all -t gfm --wrap=none \
     "../inbox/<возврат>.docx" -o "_intake-<док>-<дата>.md"
   ```
   Вставки, удаления и комментарии рецензента сохраняются в тексте
   размеченными спанами — ничего не теряется.
3. **Возврат без track changes** (правили без рецензирования, или это
   pdf/md): конвертируй как в `ingest-docs` и сравни с текущей версией
   в `spec/` — большие документы постранично через scout, не главным
   окном.
4. **Таблица решений** — в `spec/_reviews/<док>-<дата>.md`:

   | # | Место | Правка заказчика | Вердикт | Решение |
   |---|---|---|---|---|

   Вердикт по каждой правке: **принять** / **отклонить** (с причиной,
   которую увидит заказчик) / **вопрос** → строка в `QUESTIONS.md`.
   Правки заказчика — это его решения: принятые фиксируй в документе
   как «решение от <дата>» (правило 3 контракта), не растворяй молча.
5. **Внесение**: принятые правки — в целевой документ минимальным
   диффом (правило `tz-write`: документ читается целиком, чужие разделы
   не переформатируются). Строка в «Историю изменений», если документ
   её ведёт.
6. **Приёмка**: агент `verifier`, клеймы = принятые правки внесены,
   отклонённые не внесены, вопросы стоят в `QUESTIONS.md`.

## Правила

- Двухуровневость: пара правок — сразу, без служебных файлов (таблица
  решений всё равно пишется — это след согласования, не церемония);
  полный раунд согласования по большому ТЗ — `stage`.
- Файл `_intake-*.md` — рабочий полуфабрикат: после внесения решений
  его можно удалить; таблица решений остаётся.
- Конфликт правок заказчика с корпусом (противоречит соседнему ТЗ,
  глоссарию) — это не «отклонить» молча, а вопрос заказчику со ссылкой
  на источник противоречия.
