# remember — память проекта

## Что достойно записи

Ловушка, которая укусит снова; ограничение, невидимое в документах;
решение с неочевидным «почему»; проверенный приём для повторяющегося
класса задач; профиль чужой структуры ТЗ. Не достойно: то, что уже
сказано в diff/git log/корпусе, сессионная мелочь, пересказ документации.

## Записать заметку

`memory/{lessons|antipatterns|patterns|structures}/<слаг>.md`

- **lessons** — что случилось и почему так вышло.
- **antipatterns** — ловушка и как её обходить.
- **patterns** — переиспользуемый приём с 2+ реальными применениями.
  Хорошая идея с нулём применений — не паттерн: спекулятивные паттерны
  были кладбищем предшественника.
- **structures** — профиль структуры чужих ТЗ (skill `tz-adapt`).

Тело ≤ 30 строк: **Что случилось** (файл:строка, документ) →
**Почему** → **Как применять**. Связанные заметки — `[[слаг]]`.

## Проиндексировать

ОДНА строка в соответствующую секцию `memory/MEMORY.md`:

`- [Название](antipatterns/<слаг>.md) — хук для поиска`

Хук (≤120 символов) — то, по чему будущая сессия найдёт заметку
grep'ом: назови симптом, не мораль. Жёсткие правила: одна физическая
строка, без абзацев, тела в индекс не попадают никогда. Индекс
предшественника разросся до 104 КБ абзацев и стал context rot,
который каждая сессия оплачивала загрузкой.

## Поддерживать

Заметка опровергнута → обнови её на месте с пометкой, что её
заменило; дубль рядом не пишется. Секция индекса перестала читаться
одним взглядом (~40 строк) → раздели по под-доменам. Индекс работает,
только пока читается целиком.
