# knowledge-map — карта корпуса

Используй, когда корпус большой (сотни страниц) и ответ на «как у нас
устроена <тема>?» начинается с grep вслепую. Карта — навигатор:
scout, `tz-write` и владелец начинают поиск с неё, а не с полного
обхода. Первый синтез — работа уровня `stage`.

## Что строится

`wiki/_KNOWLEDGE-MAP.md` (файл без `confluence_id` — конвертер его не
трогает). Шапка: дата снапшота из `wiki/.space.json`, число доменов и
страниц, пометка «генерируется скиллом knowledge-map; факты сверяй по
[[ссылкам]] — карта навигатор, не источник истины». Затем секция на
домен:

```markdown
## <Домен>
**Что это:** 2-3 предложения.
**Вход:** [[главная страница домена]]
**Ключевое:** методы / системы / термины — списком, каждое со ссылкой.
**Читать:** 3-7 страниц в порядке погружения, [[wikilinks]].
```

## Первый синтез

1. Домены — из дерева `wiki/.pagemap.json`: страницы верхнего уровня и
   крупные ветки (~8–15 доменов). Спорную нарезку покажи владельцу в
   `brief.md` стадии до синтеза.
2. По домену — параллельный scout: читает страницы ветки, возвращает
   заготовку секции с фактами и ссылками. Главное окно только сводит —
   само страницы не читает.
3. Своди, единообразь терминологию по глоссарию, прогони `link-check`
   (карта — самый связанный файл корпуса, битая ссылка здесь стоит
   дорого).
4. Вердикт `verifier`: выборочно 2-3 домена — «Вход» существует,
   «Ключевое» подтверждается страницами, «Читать» разрешаются.

## Инкрементальное обновление — этим карта и жива

После повторного `ingest-confluence` прочитай `wiki/.ingest.json`
(added/changed/moved/removed):

- определи задетые домены по relpath изменённых страниц;
- пересинтезируй ТОЛЬКО их (scout на домен), остальные секции не
  трогай байт-в-байт;
- удалённые страницы убери из «Читать»/«Ключевое» по всем секциям
  (`link-check` подскажет, где битое);
- обнови дату снапшота в шапке.

Обновление пары доменов — второй уровень, без stage. Полный
пересинтез — только по явному запросу владельца.

## Правила

- Карта — не источник истины: факт в ТЗ цитируется со страницы-
  источника, не из карты (правило 3 контракта действует и здесь).
- Ручные правки карты умрут при пересинтезе секции — постоянные
  комментарии живут в `spec/` со ссылкой на домен.
- Не выдумывать структуру: домен без внятного содержания честно
  получает «Что это: раздел-свалка, N страниц без общей темы» — это
  находка для `tz-audit`, а не повод сочинять.
