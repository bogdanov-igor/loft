# Установка и обновление

## Требования

- [Claude Code](https://claude.com/claude-code)
- `bash`, `tar` — на macOS и Linux есть из коробки
- `python3` — все скрипты ядра только на stdlib
- Опционально: `pandoc` + `lxml` — нужны только скиллам
  `ingest-confluence` и `ingest-docs`

Рантайм-сервисы loft не нужны: ни MCP-сервера, ни демона, ни индекса,
который надо собирать. Ядро — это markdown плюс несколько детерминированных
скриптов.

## Быстрый старт

**1.** Скачайте `loft_1.1.0.tgz` и `loft_1.1.0.tgz.sha256` из
[Releases](https://github.com/bogdanov-igor/loft/releases/latest) в папку
проекта.

**2.** Откройте проект в Claude Code и скажите:

> Установи loft из архива в этой папке: сверь sha256, распакуй, запусти
> `loft/install.sh` и расскажи, что появилось.

**3.** Если проект раньше жил под specos, добавьте:

> Почисти остатки specos и предложи переаудит.

Claude сверит контрольную сумму, распакует, поставит и отчитается. Шаг
очистки уводит машинерию предшественника в карантин, ничего не удаляя — см.
[миграцию](migration.md).

## Руками

Два пути, оба запускают один и тот же инсталлер.

### Из архива

Рядом лежат два файла: `loft_1.1.0.tgz` и его сайдкар `.sha256`.

```sh
cd /path/to/project                 # оба файла скопированы сюда
shasum -c loft_1.1.0.tgz.sha256     # сначала целостность: ждём "OK"
tar -xzf loft_1.1.0.tgz
bash loft/install.sh                # без аргумента = ставим в этот каталог
```

Распакованную папку `loft/` можно оставить в проекте (её увидит проверка
обновлений, а повторный запуск `install.sh` обновит ядро) или удалить. Если
остаётся — добавьте `loft/` и `.tgz` в `.gitignore`.

### Из репозитория-исходника

```sh
git clone https://github.com/bogdanov-igor/loft.git
bash loft/install.sh /path/to/project
```

## Что делает инсталлер

| Действие | Детали |
|---|---|
| Ставит ядро | Копирует `bundle/.claude` настоящим каталогом, никогда симлинком — симлинки ломают резолв путей хуков и пер-проектность. Существующий `.claude` сначала уходит в `.claude.bak.<timestamp>`. |
| Штампует версию | Пишет `.claude/VERSION` — его читает проверка обновлений. |
| Сохраняет ваши скиллы и агентов | Каталоги скиллов и файлы агентов, которых ядро не поставляет, переносятся из прежнего `.claude`. Исключение — specos-managed `.claude`: его собственные скиллы и агенты опознаются по wire-спискам specos и остаются в бэкапе, чтобы машинерия предшественника не вернулась с переустановкой. |
| Сеет состояние проекта | Создаёт `memory/` (с `lessons/antipatterns/patterns/structures`), `stages/`, `spec/`, `inbox/`, `BACKLOG.md`, `QUESTIONS.md`, `memory/MEMORY.md` — только там, где их нет. Существующее состояние проекта не перезаписывается. |
| Защищает секреты | Добавляет `.secrets.env` в `.gitignore`. |
| Убирает MCP-налог | specos'овский `.mcp.json` (serena + playwright + memory ≈ 20–30k токенов схем на сессию) уводится в бэкап; свои серверы, если были, вы возвращаете руками. Любой другой `.mcp.json` остаётся как есть, с напоминанием: каждый сервер стоит токенов схем в каждой сессии. |
| Детектит наследие | Гоняет миграционный sweep в режиме превью и сообщает о машинерии specos/skillforge, указывая на скилл `migrate-specos`. Сам ничего не переносит. |
| Самопроверяется | Проверяет, что контракт на месте, хуки исполняемые, число скиллов сходится и `link_check` реально отрабатывает. Если что-то не так — говорит об этом громко, а не отдаёт битое ядро. |

## Обновление

Самый простой путь: скачать новый архив в папку проекта и сказать:

> Обнови loft из архива в этой папке.

Руками — та же команда, что и установка. Возьмите более новую папку loft
(скачайте релиз или `git pull`) и запустите:

```sh
cd /path/to/project
bash loft/install.sh
```

Файлы ядра заменяются. Состояние проекта — wiki, ТЗ, память, стейджи,
бэклог, вопросы, ваши собственные скиллы и агенты — не трогается. Старые
бэкапы `.claude.bak.*` можно спокойно удалять.

### Как вы узнаёте об обновлении

Хук на `SessionStart` сравнивает `.claude/VERSION` со всеми источниками,
какие видит: дистрибутив loft рядом (папка `loft/` в проекте или
`$LOFT_HOME`) и последний [релиз на GitHub](https://github.com/bogdanov-igor/loft/releases) —
и печатает одну строку, когда что-то строго новее. Если версия актуальна, не
печатает ничего, так что обычный случай стоит ноль токенов.

GitHub-запрос кэшируется на 24 часа и имеет жёсткий потолок 3 секунды —
медленная сеть не задержит старт сессии; любой сбой — тихий выход. Полное
отключение: `LOFT_NO_UPDATE_CHECK=1`. Заблокировать сессию проверка не может.

## Сборка архива (мейнтейнерам)

```sh
bash build-archive.sh    # → dist/loft_<version>.tgz + .sha256
```

`build-archive.sh` в архив не кладётся. Сначала он прогоняет `test/run.sh` —
48 самотестов скриптов ядра, офлайн, на одноразовых фикстурах — и
отказывается собирать, если хоть что-то упало. Затем распаковывает только что
собранный архив во временный каталог и делает настоящую установку, проверяя
результат от начала до конца.
