# Changelog

## [1.2.0] — 2026-07-15

- Скраб атрибутов в конвертере (clean_dom, вайтлист по тегам): Confluence
  вешает class/style/rel/data-*/aria-* на всё подряд, а pandoc сохраняет
  любой инлайн-элемент с атрибутами сырым HTML — из-за этого ссылки в
  потоке оставались `<a class=...>` вместо `[текст](url)` → `[[wikilink]]`.
  Теперь у `<a>` остаются только href/title, у `<img>` — src/alt/width/
  height, у `<td>/<th>` — colspan/rowspan, у `<pre>/<code>` — class (язык
  фенса). Fallback-таблицы сериализуются без шлака (`<td>` вместо
  `<td class="confluenceTd" style=...>`). На реальном корпусе (336 стр.):
  665 HTML-якорей в потоке → 0.
- Декоративные иконки выпиливаются: jira-аватарки (viewavatar/useravatar),
  эмотиконы — внешние `<img>`, привязывавшие страницы к живому трекеру.
- Автолинки: `<a href="X">X</a>` (текст == URL) после скраба становится
  `<url>`-автолинком; post_process теперь резолвит такие в `[[wikilink]]`
  (или локальный файл) так же, как обычные ссылки. Фенс-безопасно.
- fix_tables v2 — постпроцессор доводит старую wiki до вывода нового
  конвертера: выпил иконок, конверсия простых `<a>` в потоке в markdown/
  wikilinks (внутренние `.md` → `[[base|текст]]`, вложения → `[[файл|имя]]`,
  внешние → `[текст](url)`, текст==URL → `<url>`), скраб атрибутов
  оставшихся тегов (в т.ч. внутри fallback-таблиц), `[текст](x.md)` →
  wikilinks. Фенсы, инлайн-код и экранированные литералы (`\<b\>`)
  неприкосновенны; идемпотентно; печатает счётчик wikilinks до/после.
- `--expand-spans` в convert.py и fix_tables.py: таблицы с colspan/rowspan
  не остаются HTML-fallback'ом, а разворачиваются в GFM — rowspan повторяет
  значение на каждой строке, colspan дополняет пустыми ячейками (потеря
  вёрстки, не данных). Флаг опциональный: меняет представление объединённых
  ячеек, решение за владельцем корпуса.
- Находки полного аудита корпуса (338 стр., 14 scout'ов + адверсариальная
  верификация каждой категории двумя агентами против исходной выгрузки):
  - ссылки формата `/spaces/<KEY>/pages/<id>` теперь резолвятся в wikilinks;
  - декодер tiny-ссылок `/x/<code>` чинён A-паддингом коротких кодов
    (проверено на ground-truth паре WooqB ↔ 69896794);
  - нормализация emphasis до pandoc: хвостовой `<br>` и граничные пробелы
    выносятся из `<strong>/<em>`, вложенный болд сплющивается — уходит битый
    GFM вида `**X\**` и `**X **и**Y**`;
  - переэкранированные стрелки pandoc `-\>` → `->` (фенсы и инлайн-код
    неприкосновенны);
  - дефолтный плейсхолдер expand-макроса «Нажмите здесь для раскрытия...» /
    "Click here to expand..." → «Подробнее»;
  - unknown-macro (макрос, который экспорт не отрендерил) оставляет видимый
    маркер вместо молчаливого удаления;
  - многострочные fallback-таблицы схлопываются в одну строку (перенос
    внутри raw-HTML обрывает блок у md-рендереров), кроме таблиц с `<pre>`.
- fix_tables: те же ремонты для уже сконвертированного корпуса + при наличии
  `wiki/.pagemap.json` абсолютные confluence-URL (pageId, /spaces/, /x/tiny)
  резолвятся в wikilinks (в потоке) и относительные пути (в raw-таблицах);
  артефакт `[!KEY](jira-url)` теряет лишний `!`.
- install.sh: самопроверка не валится на битых ссылках в spec живого проекта
  (link_check честно выходит с кодом 1 — это не поломка скрипта).
- test/run.sh: 102 самотеста.

## [1.1.0] — 2026-07-15

- `templates.md` в tz-elicit: шаблоны формулировок требований — EARS
  по-русски (всеобщее/событийное/состояние/нежелательное/опциональное),
  user story с критериями Given-When-Then, таблица НФТ с колонкой «как
  измеряем». tz-write ссылается на них в шаге черновика.
- Новый скилл knowledge-map (14-й): воспроизводимая карта корпуса —
  навигатор по доменам wiki (что это / вход / ключевое / читать), синтез
  параллельными scout'ами. После обновления снапшота пересинтезируются
  только домены, задетые `_CHANGES`.
- `--unroll-pre` в convert.py и fix_tables.py: таблицы с многострочными
  JSON/XML-примерами разворачиваются — в ячейке остаётся «см. Пример N
  ниже», код выносится после таблицы фенсами с языком (из class или
  детерминированного сниффа), содержимое байт-в-байт. На реальном корпусе:
  140 таблиц в 109 файлах ушли в чистый GFM, идемпотентно, wikilinks не
  тронуты. Флаг опциональный, потому что меняет вёрстку страницы; без него
  поведение прежнее.
- test/run.sh: 48 самотестов (добавились unroll и провязка templates).

## [1.0.0] — 2026-07-15

Первый публичный релиз / first public release —
github.com/bogdanov-igor/loft.

Ядро документной работы для Claude Code:

- Контракт на 81 строку (~3k токенов always-on): профессия
  системного/бизнес-аналитика и указатели; «ни одного выдуманного факта»;
  правило границы — скрипт в ядре только детерминированный конвертер,
  проверки смысла — указания агентам.
- 13 lazy-скиллов: tz-write, tz-adapt, tz-audit, tz-elicit (банки вопросов
  по 6 типам задач), ingest-confluence, ingest-docs, review-intake (возврат
  заказчика с track changes — правки не теряются), link-check, deliver-pdf
  (цепочка движков typst → weasyprint → wkhtmltopdf → Chrome headless →
  HTML), spec-bootstrap, remember, stage, migrate-specos (карантин машинерии
  предшественника с манифестом отката).
- Конвертер Confluence: детерминированный (pandoc + lxml, без LLM-рерайта),
  собственный писатель таблиц HTML→GFM с логом fallback'ов, имена вложений
  из alias, stale-removal с защитой рукописных файлов, `--space` /
  `--base-url`, отчёт изменений при повторном ingest (`_CHANGES-<дата>.md` +
  `.ingest.json`, sha1-хэши без ложных срабатываний). Обкатан на реальном
  корпусе: 446 страниц, 0 битых из 3645 ссылок, дерево байт-в-байт с
  эталоном.
- 2 сабагента (scout, verifier), 2 хука (leak-guard, update-check: локальные
  источники + GitHub-релизы, кэш 24 ч, тихий), файловая память,
  BACKLOG/QUESTIONS.
- Жизненный цикл: install.sh с самопроверкой и бэкапами; обновление той же
  командой; корпус — граф Foam/Obsidian из коробки (инсталлер сеет
  рекомендацию расширения); 40+ самотестов как релизный гейт сборки;
  golden-фикстуры экспорта Confluence.
- Документация EN+RU внутри архива, Apache-2.0.
